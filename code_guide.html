Chapter 1 · index.html
The Blueprint 🏛️
process_dashboard/index.html
HTML is the skeleton of every webpage. Every element you see — buttons, charts, cards — starts as an HTML tag. This file defines the structure of the entire dashboard: 5 views, a sidebar, charts, and forms.

🧠 Before You Start: What is HTML?

HTML = HyperText Markup Language. It uses <tags> to describe content. Think of it like a Word document's structure — headings, paragraphs, tables — except browsers understand it. Each <tag> has an opening <div> and closing </div>. Tags can have attributes like id="myButton" or class="card" which CSS/JS use to style/control them.

Part 3 — The SPA Views (Single-Page Application Pattern)

🧠 Key Concept: Single-Page Application (SPA)

Traditional websites load a new HTML file for every page. SPAs load ONE HTML file and use JavaScript to show/hide different sections. This is how Gmail, Twitter, and most modern apps work. Our 5 "pages" are actually 5 <section> elements — only one has class="view active" at a time.

🧠 What is an Object in JavaScript?

An object is a collection of related data and functions grouped together. Instead of having 20 separate variables and functions flying around, we group them in one object: const DataGenerator = { ... }. Functions inside objects are called "methods". Access them with: DataGenerator.generateEventLog().

The path [0,3,3,4...] has activity 3 twice — that's a rework loop (inventory check failed, try again)!

Math.random() = random decimal between 0 and 1.
× 90 × 24 × 60 × 60 × 1000 = multiply by milliseconds in 90 days (90 days × 24 hours × 60 min × 60 sec × 1000 ms).

Lines 64–65

this._normalRandom() = returns a random number from a bell curve (mostly near 0, rarely far from 0).
× activity.stdDev = scale the randomness by the standard deviation.
+ activity.avgDuration = shift to center around the average.
Math.max(0, ...) = wait times can't be negative! This clamps to minimum 0.

Example: avgDuration=8, stdDev=6 → wait times are mostly 2–14 hours, rarely up to 20+.
Line 67

Date arithmetic. getTime() converts a Date to milliseconds since Jan 1, 1970 (Unix timestamp). Add the wait hours converted to milliseconds. Wrap in new Date() to convert back. This advances the clock by the wait time.
Lines 69–74
events.push({ case_id, activity, timestamp, resource })
events.push() adds an item to the end of the array. We push an object literal — a new event row. This is one row in our event log table. After all loops complete, events[] contains thousands of these row objects.
Line 77

Array.sort() with a comparator. Without a comparator, sort() converts to strings (wrong for dates!). With (a, b) => a.timestamp - b.timestamp: negative result = a comes first, positive = b comes first, zero = equal. This sorts chronologically (oldest event first).
Part 3 — The Box-Muller Transform (Statistics Algorithm)
data_generator.js — _normalRandom()
90 _normalRandom() {
91 let u = 0, v = 0;
92 while (u === 0) u = Math.random();
93 while (v === 0) v = Math.random();
94 return Math.sqrt(-2.0 * Math.log(u)) * Math.cos(2.0 * Math.PI * v);
95 },
Lines 91–95
Box-Muller Transform — a mathematical algorithm to generate normally distributed (bell curve) random numbers from uniform random numbers.

Math.random() gives uniform randomness (equal chance of any value). But real-world durations follow a bell curve — most close to average, few extremes.

Lines 92–93: Loop until we get non-zero values (avoids log(0) = -Infinity error).
Line 94: The formula transforms two uniform randoms (u, v) into one normally distributed random. The math uses natural log and cosine — trust the formula, it's proven statistics.
You don't need to understand the math. The important thing: this gives us realistic bell-curve distributed numbers, making our fake data look like real business data.

Line 95

Gets the last element of an array. Arrays are 0-indexed (first = [0]). Last = [length - 1]. For an array of 9 events, last index = 8 = [9-1]. This gives us the final event's timestamp = when the case ended.
Line 96
caseData.duration = (end - start) / (1000 * 60 * 60)
Date subtraction gives milliseconds. Divide by 1000 → seconds. Divide by 60 → minutes. Divide by 60 → hours. So this converts the duration from milliseconds to hours.
Part 3 — _detectBottlenecks() : The Core Algorithm
process_engine.js — _detectBottlenecks()
160 _detectBottlenecks(graph, activityStats) {
161 const bottlenecks = [];
163 let maxWait = 0, maxFreq = 0;
164 for (const [, stat] of activityStats) {
165 maxWait = Math.max(maxWait, stat.avgWait);
166 maxFreq = Math.max(maxFreq, stat.frequency);
167 }
169 for (const [activity, stat] of activityStats) {
170 const normalizedWait = stat.avgWait / maxWait;
171 const normalizedFreq = stat.frequency / maxFreq;
172 const score = normalizedWait * 0.7 + normalizedFreq * 0.3;
174 bottlenecks.push({ activity, score, severity: score > 0.7 ? 'critical' : 'high' });
175 }
177 bottlenecks.sort((a, b) => b.score - a.score);
178 return bottlenecks;
179 },
Lines 163–167
Finding maximums. Before scoring, we need to know the maximum wait time and maximum frequency across all activities. This is used for normalization.

SVG = Scalable Vector Graphics. Unlike PNG/JPEG (pixel images), SVG uses math to describe shapes. A circle in SVG: <circle cx="50" cy="50" r="20">. Zooming in never pixelates. D3.js generates SVG elements programmatically from data — "data-driven documents" (that's what D3 stands for!).

Part 3 — Topological Layout Algorithm
process_map.js — _computeLayout() key parts
110 const inDegree = new Map();
111 for (const [name] of nodes) inDegree.set(name, 0);
112 for (const edge of edges) {
113 inDegree.set(edge.target, (inDegree.get(edge.target) || 0) + 1);

116 const queue = [];

118 if (deg === 0) { queue.push(name); nodeLayer.set(name, 0); }
119 }
121 while (queue.length > 0) {

123 const successors = adjList.get(current) || [];
124 for (const next of successors) {
125 const nextLayer = Math.max(nodeLayer.get(next) || 0, currentLayer + 1);
126 nodeLayer.set(next, nextLayer);

128 }
129 }
Lines 110–114
inDegree: how many arrows point INTO each node
In-degree = number of incoming edges to a node. "Order Received" has 0 incoming edges (nothing comes before it — it's the start). "Credit Check" has 1 incoming edge (from "Order Received").

We first initialize all to 0, then for each edge, increment the target node's count. This is like counting "how many prerequisite activities must happen before this one?"
Lines 117–119
if (deg === 0) → put in layer 0 (start of queue)
Finding start nodes: nodes with in-degree 0 have no predecessors — they're process starts. These go in Layer 0 (leftmost column). "Order Received" → Layer 0.
Lines 121–129
BFS — Breadth-First Search
Breadth-First Search (BFS) — a graph traversal algorithm that processes nodes level by level.

queue.shift() = removes and returns the FIRST item (FIFO: First In, First Out — like a real queue/line at a shop).
For each node we process, we find its successors and assign them to the next layer (current + 1).
Math.max(...) handles cases where a node has multiple paths leading to it — it goes to the DEEPEST layer required.
Imagine water flowing through a network of pipes. BFS is like water spreading out layer by layer — all nodes at depth 1 fill before depth 2, etc.

Part 3 — Counter Animation (requestAnimationFrame)
dashboard.js — animateCounter()
200function animateCounter(elementId, target) {
201 const el = document.getElementById(elementId);
202 const duration = 800;
203 const start = performance.now();
205 const tick = (now) => {
206 const elapsed = now - start;
207 const progress = Math.min(elapsed / duration, 1);
208 const eased = 1 - Math.pow(1 - progress, 4);
209 const current = Math.round(target * eased);
210 el.textContent = current.toLocaleString();
211 if (progress < 1) requestAnimationFrame(tick);
212 };
213 requestAnimationFrame(tick);
214}
Line 203
performance.now()
Returns the current time in milliseconds with sub-millisecond precision. Unlike Date.now(), it's not affected by system clock changes. Used to measure elapsed time since animation started.
Lines 205–213
requestAnimationFrame(tick)
requestAnimationFrame (rAF) — asks the browser to call a function just before the next screen repaint (typically 60 times per second = 60fps).

The tick function is called repeatedly:
1. Calculate how much time has passed (elapsed)
2. Calculate progress as 0→1 (elapsed / total_duration)
3. Apply easing formula to make it feel smooth
4. Update the number on screen
5. If not done yet, schedule the next frame
6. When progress reaches 1 (100%), stop
Like a flip-book animation: each frame is one "tick". Show frame 1, then frame 2, then frame 3... at 60fps it looks smooth.
Line 208
eased = 1 - Math.pow(1 - progress, 4)
easeOutQuart easing. Without easing, animation moves at constant speed (linear). Easing makes it start fast and decelerate naturally.

At progress=0: eased = 1 - (1)^4 = 0 (start)
At progress=0.5: eased = 1 - (0.5)^4 = 0.9375 (90% done at halfway!)
At progress=1: eased = 1 - 0 = 1 (end)

The counter races up quickly then slows down — feels snappy and satisfying!
